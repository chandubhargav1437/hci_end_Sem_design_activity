Frequently Asked Questions

Go to Services → Download e-Aadhaar → Select format (Standard PDF recommended) → Enter OTP → Download. The password is first 4 letters of name (uppercase) + year of birth. Example: RAHU1995

Go to Services → Update Address → Fill new address → Upload a valid Proof of Address document → Verify with OTP → Submit. Processing takes 5–7 working days. You will receive SMS confirmation.

VID is a 16-digit temporary number that can be used instead of your actual Aadhaar number for authentication. It protects your real Aadhaar number. Generate VID from Services → Generate VID. It can be regenerated every 24 hours.

Go to Profile → Security Settings → toggle "Biometric Lock". When locked, fingerprint and iris-based authentication is disabled, preventing unauthorized use. You can unlock it anytime with OTP verification.

Possible reasons: Biometrics locked, fingerprint mismatch (try cleaning fingers), expired OTP, or service outage. Check Auth History for error codes. If biometrics fail repeatedly, visit nearest Aadhaar Enrollment Centre for re-enrollment.

🧠 HCI Law Mapping

Every UI element mapped to its governing HCI principle — for evaluation & viva

Pareto 80-20
Top 3 Quick Actions on dashboard: Download, Update Address, Auth History — 80% of user needs served by 20% of features. Positioned prominently below the Aadhaar card.
Miller's Law 7±2
Services grid has exactly 8 icons (≤9). Bottom navigation has 4 items. FAQ section has 6 questions. Each filter chip row ≤6 items to avoid cognitive overload.
Hick's Law
OTP login has only 2 inputs. Download screen offers 3 format choices. Filter chips reduce visible list items. Every decision point minimized to ≤3 options where possible.
Fitts' Law
All primary buttons are full-width (54px+ height). Quick action cards are large tap targets. Bottom nav items are ≥62px tall. OTP cells are 48×56px — large for thumb reach.
Serial Position
Primacy: Download, Update, History appear first (most critical). Recency: Logout placed last in Settings. Auth history sorted newest first. FAQ most-asked question is first.
Zipf's Law
Most-Used section in Services lists top 3 services. Download e-Aadhaar appears on Home AND Services. Address Update is accessible from 3 paths. Frequent features have shortcuts.
Jakob's Law
Design mirrors DigiLocker/UMANG apps: bottom navigation, card-based layout, status bar at top, avatar + name in header — familiar patterns reduce learning curve.
Norman Principles
Visibility: Status badges everywhere. Feedback: Toast notifications on every action. Constraints: PIN limited to 4 digits, mobile to 10 digits. Mapping: Green=success, Red=error. Affordance: Buttons look tappable with shadows.
Shneiderman 8 Rules
(1) Consistency: same card style everywhere. (2) Shortcuts: biometric login, quick actions. (3) Feedback: toast + badges. (4) Dialogs: 4-step address wizard. (5) Error prevention: constraints + alerts. (6) Reversal: Reset + Cancel buttons. (7) User control: lock/unlock toggles. (8) Memory load: Recognition over recall throughout.
Nielsen Heuristics
(1) System status: verified badge, biometric dots. (2) Real world: "Aadhaar card" card design. (3) User control: back button on every screen. (4) Consistency: blue=interactive, green=ok. (5) Error prevention: OTP before sensitive actions. (6) Recognition: icons with labels. (7) Flexibility: biometric + OTP + password.
Gestalt Laws
Proximity: Biometric cards grouped 2×2. Similarity: All service items same card style. Closure: Step wizard shows incomplete circle → motivates completion. Common Region: Profile sections in bordered cards. Figure-Ground: Modal overlays. Continuity: Progress bar flows left-to-right. Symmetry: Stats summary in 3-column grid.
Inverted Pyramid
Dashboard layout: Aadhaar card + verification status (most critical) → Top 3 quick actions → 8 service grid → Recent activity (detail). Most important information always at the top.
Tesler's Law
Complexity conserved, not eliminated: Address update has 4 steps (document upload, form, review, submit) because the complexity is inherent. The UI absorbs it via wizard pattern so user effort is minimized.
Power Law of Practice
Biometric login shortcut on login screen. Quick Actions at top of dashboard. Frequent services appear before alphabetical list. Returning users skip onboarding directly to auth.
Asimov UX Safety
Never harm user data: Biometric lock toggle prevents unauthorized auth. Screenshot protection prevents data leakage. OTP required before download. Unauthorized attempts are blocked and logged.
Persuasion Psych
Authority: UIDAI emblem, Govt. of India branding, ISO badge. Social Proof: "1.3B+ Users" on splash. Trust: Blue color palette, verified checkmarks. Scarcity: "OTP expires in 30s" urgency. Reciprocity: Free download emphasized.
Color Psychology
Blue (#2563c8): Trust, government authority — primary brand color. Green (#22c55e): Success, verified, safe. Red (#dc2626): Errors, failures, warnings. Amber (#f59e0b): Caution, pending. Saffron (#FF6B00): Indian identity, accent.
Universal Design
9 Indian languages in settings. Voice assistance toggle. High contrast mode. Large text (1.3× scale). Haptic feedback. All tap targets ≥44px (WCAG AA). F-pattern layout with left-aligned text.
Alphabetical Sorting
State dropdown is alphabetically sorted (Andhra Pradesh → West Bengal). Services search results sort alphabetically. Filter chips sort All→Identity→Updates→Security→Downloads (frequency then alpha).
Password Masking
Aadhaar number masked by default (shows XXXX XXXX ####). Eye toggle reveals full number. Share code input is type="password". OTP cells show ● when filled. Unmask button always visible per W3C guidelines.
Law of Learning
Onboarding splash explains app identity. First-time hints on OTP screen. FAQ in Help covers all common operations. Consistent placement means users quickly memorize navigation paths.
Aesthetic Usability
Gradient hero sections appear more premium and trustworthy. Card shadows add depth. Indian tricolor strip on splash communicates legitimacy. Professional blue/white palette increases perceived reliability.
Principle of Least Effort
Download accessible in 2 taps from dashboard. Address update pre-fills known data. Biometric login skips all manual entry. Search in Services avoids scrolling. Most-used section avoids hunting.
Mental Models
Aadhaar card widget resembles physical Aadhaar card with masked number. Document upload resembles file picker. Step wizard resembles form filling at service centre. Users map digital → physical experience.
